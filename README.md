# 放置天堂 ⚡ 一鍵外掛

為「放置天堂」網頁版遊戲設計的 Tampermonkey 單一腳本外掛。功能集中於同一份使用者腳本，提供加速、背包管理、物品資訊、協力隊伍、城堡快捷、角色調整、自動保命及多項介面優化。

目前穩定版本：`v3.29.4`

[一鍵安裝外掛](https://raw.githubusercontent.com/qcc781192000/idle-lineage-plugin/main/main.user.js)｜[版本發行頁](https://github.com/qcc781192000/idle-lineage-plugin/releases)｜[問題回報](https://github.com/qcc781192000/idle-lineage-plugin/issues)

## 安裝需求

- Chrome、Edge、Firefox 或其他可安裝 Tampermonkey 的瀏覽器。
- 已安裝並啟用 [Tampermonkey](https://www.tampermonkey.net/)。
- 使用支援的放置天堂遊戲頁面。

## 安裝方式

1. 在瀏覽器安裝並啟用 Tampermonkey。
2. 使用同一個瀏覽器開啟[一鍵安裝連結](https://raw.githubusercontent.com/qcc781192000/idle-lineage-plugin/main/main.user.js)。
3. 確認 Tampermonkey 顯示腳本安裝頁。
4. 按下「安裝」，再重新整理遊戲頁面。

若瀏覽器只顯示程式碼，請確認連結最後是 `.user.js`，而且目前瀏覽器已啟用 Tampermonkey。

## 自動更新

腳本標頭已設定 GitHub 主分支的 `@updateURL` 與 `@downloadURL`。Tampermonkey 會依自己的更新週期檢查版本，只有 `@version` 較新的版本才會套用。

手動檢查更新：

1. 開啟 Tampermonkey 管理面板。
2. 找到「放置天堂 ⚡ 一鍵外掛」。
3. 執行「檢查更新」。

腳本名稱及 `@namespace` 維持不變，因此正常更新不會建立第二份外掛。

## 主要功能

### 加速與效能控制

- 最高 `50000×` 目標倍率與常用倍率快捷鍵。
- 嚴格流暢、平衡及吞吐優先三種效能模式。
- 顯示目標倍率、實際倍率與裝置負載狀態。
- 可拖曳的加速控制鈕及自訂熱鍵。
- 共用刷新排程，避免各模組重複建立高頻計時器。

### 自動保命

- 自動復活、瞬間移動與回村控制。
- 依生命值門檻判斷保命動作。
- 保留手動開關及設定記憶。
- 共用排程器限制重複觸發。

### 武器、防具與道具管理

- 武器、防具及道具各自保留分頁狀態。
- 依可用高度自動計算每頁數量。
- 名稱、分類及屬性篩選。
- 快速強化、快速廢品與跨頁批次選取。
- 「全選」作用於全部符合目前篩選條件的物品。
- 背包內容減少時自動修正到有效頁碼。
- `ESC` 依浮動層、快速模式及一般選取的優先順序逐層取消。

### 統一物品資訊

- 武器、防具、道具、裝備、遺物及寵物裝備支援懸停資訊。
- 顯示完整名稱、數量、鎖定、強化、主要能力及特殊詞綴。
- 遺物顯示特殊效果與完整機制。
- 提示框依視窗邊界自動調整方向。
- 長內容可捲動，不會裁切底部資訊。
- 遺物完整資訊視窗支援桌面雙欄及窄畫面上下排列。

### 協力隊伍與寵物

- 傭兵、寵物及召喚物分頁管理。
- 出戰寵物固定排在名冊最上方，但不使用懸浮定位。
- 其餘寵物預設收合，展開狀態會保存在瀏覽器。
- 其餘寵物原地展開／收合，不重建整個寵物面板。
- 其他角色出戰寵物排列在保管寵物之前。
- 寵物武器、防具、出戰、召回、鎖定、進化及放生操作。
- 出戰、召回或轉移後立即重新分組。
- 倒地數量只在摘要顯示一次，避免重複提示。

### 城堡、血盟與快捷面板

- 從城堡狀態卡選擇肯特、風木或海音城。
- 城堡切換沿用正式攻城流程，不直接改寫城堡所有權。
- 保留等級、搜索狀、冷卻、攻城中及待領賞限制。
- 可從野外或地監直接開啟城堡護衛及魔物追蹤。
- 城堡護衛限定實際擁有的城池。
- 血盟面板保留加入、祝福、攻城與領賞功能。
- 城堡、血盟及黑市快捷視窗由共用浮動層管理。

### 角色與裝備調整

- 六項基礎能力調整。
- 技能啟用與停用控制。
- 裝備屬性安全調整。
- 空裝備欄精確選擇器。
- 背包物品數量調整。
- 職業專精徽章顯示與自訂。

### 福利倍率

- 玩家、傭兵、寵物及召喚物共用來源適配器。
- 經驗、金幣、掉落、傷害、恢復、攻速及施法速度等倍率。
- 傭兵物理、魔法及持續傷害減免。
- 卡片與祝福卷軸掉落倍率。
- 新增來源時不需要重複包裝整套戰鬥函式。

### 其他介面功能

- 收藏冊項目開關與狀態記憶。
- NPC 製作材料來源及快速前往。
- 系統日誌旁的潘朵拉商品櫥窗。
- 遊戲模式切換與模式收藏快照。
- 傭兵能力及成長資料完整性修復。
- Tooltip、確認視窗、物品視窗與快捷面板的共用層級管理。

## 設定與資料

- 外掛設定儲存在瀏覽器 `localStorage`。
- 外掛不需要獨立伺服器或資料庫。
- 清除網站資料會一併清除外掛設定。

## 鍵盤與操作規則

- 加速開關可在外掛面板設定熱鍵。
- `ESC` 每次只處理最上層視窗或目前頁面的最高優先狀態。
- 懸停物品只顯示資訊，不會觸發裝備、販售、強化或使用。
- 快速操作模式啟用時，點擊物品仍以勾選為優先。

## 疑難排解

### 點擊安裝連結沒有出現 Tampermonkey

- 確認使用已安裝 Tampermonkey 的瀏覽器開啟連結。
- 確認安裝網址最後為 `main.user.js`。
- 確認 Tampermonkey 擴充功能已啟用。
- 仍無法安裝時，可在 Tampermonkey 管理面板選擇「從網址匯入」，再貼上一鍵安裝網址。

### 更新後功能沒有變化

- 在 Tampermonkey 管理面板確認版本號。
- 手動執行「檢查更新」。
- 重新整理遊戲頁面。
- 確認沒有同時啟用舊版拆分腳本。

## 開發與發布規範

1. 所有功能只修改根目錄 `main.user.js`，不得修改 `Source/`。
2. 不得建立第二份正式腳本或功能副本。
3. 每次新增功能或修正使用者可見問題，都必須提升 `@version`。
4. 發布前至少執行 JavaScript 語法檢查及對應功能驗證。
5. 只有通過驗證的版本可以推送到 `main`。
6. 每次發布都必須提交 Git、建立版本標籤並推送 GitHub。
7. 每個 GitHub Release 都必須清楚列出新增、調整、修正及驗證內容。
8. 不得提交 `Source/`、本機憑證、暫存檔、代理設定或測試工作資料。
