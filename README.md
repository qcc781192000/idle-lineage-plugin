# 放置天堂 ⚡ 一鍵外掛 — Tampermonkey 單一腳本

## 📦 檔案

| 檔案 | 說明 |
|------|------|
| `tampermonkey/afk-main.js` | ⚡ **單一外掛腳本** — 所有功能整合於此檔案，後續擴充皆在此 |
| `tampermonkey/afk-main.user.js` | Tampermonkey 安裝入口，內容與 `afk-main.js` 完全相同，不是另一份外掛 |
| `tampermonkey/README.md` | 本說明文件 |
| `tampermonkey/test.txt` | 參考用（不更動） |

## 規則

1. **所有功能只寫在 `afk-main.js`**，不另開新腳本。
2. **只修改 `tampermonkey/` 目錄內的檔案**，**禁止改變 `Source/` 目錄內任何資料**（包括 `js/`、`css/`、`index.html` 等遊戲原始碼與專案檔案）。

	**強制限制（必須遵守）**: 本專案外掛維護政策要求所有程式修改與功能新增僅限於 `tampermonkey/` 目錄，嚴禁變更 `Source/` 目錄內任何資料。任何需要變更遊戲核心檔或專案原始碼的提案均不予允許。外掛端必須實作必要的 fallback 或映射機制（例如本腳本的 row→inv fallback、DOM data-attributes 補丁）以避免直接變更 `Source/` 目錄內之遊戲程式碼。
3. 各模組以 IIFE 封裝，獨立命名空間。
4. 所有設定存 `localStorage`，key 前綴 `afk_`。
5. 後續新增功能請在 `afk-main.js` 下方新增模組區塊，遵循相同模式。

## 🚀 安裝方式

1. 瀏覽器安裝 [Tampermonkey 擴充功能](https://www.tampermonkey.net/)
2. 點擊 **[安裝放置天堂一鍵外掛](https://raw.githubusercontent.com/qcc781192000/idle-lineage-afk-plugin/main/afk-main.user.js)**
3. Tampermonkey 顯示安裝頁後按下「安裝」
4. 進入放置天堂遊戲即可使用

### 自動更新

- 腳本會透過 GitHub 主分支的 `afk-main.user.js`、`@updateURL` 與 `@downloadURL` 自動檢查穩定版本。
- `afk-main.user.js` 採用 Tampermonkey 可直接辨識的副檔名，安裝後仍只會顯示原本的「放置天堂 ⚡ 一鍵外掛」。
- 只有完成驗證並提升版本號的內容才會發布到主分支。
- 如要立即檢查，開啟 Tampermonkey 管理面板，在本腳本選單執行「檢查更新」。
- 腳本名稱與命名空間維持不變，從舊版升級不會建立第二份腳本。

> ⚠️ 如果之前有安裝過舊版（`afk-speed.js` / `afk-itemsearch-plus.js` / `afk-revive.js`），請先移除那些舊腳本，只保留 `afk-main.js`。

## ⚡ 功能總覽

### 模組一：加速器（⚡ 浮動面板）

| 功能 | 說明 |
|------|------|
| **多倍加速** | 2× / 5× / 10× / 50× / 100× / 500× + 滑桿微調至 2000× |
| **安全回村** | HP 低於設定值自動回村 |
| **自動瞬移** | HP 更低時優先瞬移躲避怪物 |
| **自動復活** | 死亡後自動原地復活 |
| **浮動面板** | ⚡ 圖示拖曳、懸停展開、可釘選 |
| **熱鍵 F6** | 一鍵切換加速開關 |

| 功能 | 說明 |
|------|------|
| **搜尋列固定** | 永遠在最上方，不隨內容捲動 |
| **分類下拉** | 全部 / 祝福的 / 詛咒的 / 傳說 / 遠古 / 強化+5↑ / 未強化 / 遺物 |
| **雙重過濾** | 名稱 + 分類 AND 過濾 |

### 模組三：復活開關（🔄 畫面按鈕）

| 功能 | 說明 |
|------|------|
| **開關按鈕** | 遊戲畫面左下角 🟢 / 🔴 點擊切換 |
| **自動復活** | ON 時死亡自動原地復活，2 秒冷卻 |
| **記憶設定** | localStorage 記住開關狀態 |

## ⚙️ 加速原理

加速器包裝 `gameLoop`，在每次呼叫時注入 `(speedRate-1)*100ms` 到 `_tickDebt`，讓遊戲 tick 累積更快 → 自然觸發更多 tick/秒。**不碰 `performance.now`、`Date.now`、`requestAnimationFrame`、`setTimeout`**，渲染與動畫完全不受影響。
